> 说在前头：生命不息，折腾不止。作为一个重度电子产品和代码折腾爱好者，看到博客哪里不顺眼就总想改改。今天终于把首页那个个人介绍卡片爆改了，顺便给整个站加上了评论区！

## 一、个人卡片大翻新：RPG 玩家尊享版

之前博客的个人卡片还是比较素的，在宽屏桌面端甚至还会发生错位的问题。今天花了点时间彻底重构了一下 UI。

- **动态呼吸灯**：在头像右下角加了个绿色的呼吸状态灯，旁边写着 `CODING`，感觉就像是我 24 小时高强度在线（虽然实际上可能在睡觉 /doge/）。
- **游戏化面板**：结合我的十年游戏史，把那些略显无聊的“高一学生”、“熟悉各种系统开关机”改成了 RPG 游戏面板里的元素。比如一个 `Lv.16 | EXP 85%` 的经验值条，以及把平时折腾的软件和代码强行塞进了 `ATK`（攻击力）、`INT`（智力）和 `AGI`（敏捷）的属性徽章里。中二魂直接拉满！
- **移动端专属优化**：顺带把移动端的卡片顺序提到了文章列表上面，方便大家一进来就能看见我炫酷的面板。

## 二、重头戏：引入 Giscus 评论系统

博客建好之后，怎么能没有评论区呢？
对比了一圈 Waline、Twikoo 和 Disqus 之后，我果断选择了 **Giscus**。

原因很简单：**完全白嫖、没有广告、纯静态加载不拖慢网速**，而且它是借助 GitHub Discussions 来存评论的。虽然这也意味着评论的人必须要有 GitHub 账号，但对于会来看我这个极客博客的人来说，谁还没个 GitHub 账号呢？

### 配置过程（看似有手就行）

其实 Giscus 的配置特别无脑：
1. 保证你的仓库是 Public（公开）的。
2. 在仓库的 Settings 里把 Features -> **Discussions** 勾上。
3. 去 Giscus 官网，把你的仓库名（比如 `GreenRoc114/Blog-reborn`）填进去，选择映射关系和类别。
4. 拿到它生成的那段 `<script>` 代码，扔进你的页面里。

为了适配本站用的 `swup` 单页应用跳转（SPA 无刷新加载），我还专门写了事件监听器去重新挂载 Giscus 脚本，避免跳转下一篇文章时评论区卡死。

**本来以为这就完事了，结果刷新一看，评论区压根出不来，直接报错！**

## 三、我踩的那个坑：忘记装 Giscus App！

我在浏览器控制台抓耳挠腮半天，翻来覆去检查了好几遍我的代码和 Repo ID。后来仔细一查，发现官方文档写得清清楚楚，Giscus 正常工作必须满足 **三个条件**：

1. **该仓库是公开的**，否则访客将无法查看 discussion。（这条我满足了）
2. **giscus app 已安装**，否则访客将无法评论和回应。（🚨 **就是这条没搞！**）
3. **Discussions 功能已在你的仓库中启用**。（这条也满足了）

我满脑子都是去开 Discussions 功能和生成代码，居然完全忘记了去 GitHub 市场里安装 Giscus 的授权 App！没有给它授权，它哪来的权限去帮你操作 Discussions 发评论啊喂！

**解决方法很简单：**
去 GitHub Apps 里搜索 **giscus**，点击 `Install`，然后选择给 `Blog-reborn` 仓库授予权限就行了。

授权完再去刷新页面，评论区瞬间丝滑加载，舒服了。
